## 0. 이 문서의 범위와 확정 사항

이번 개발 단계는 **데이터 수집·디자인 이전에, 결정론 연산 엔진(=서비스의 '두뇌')을 mock 데이터로 완성·검증**하는 것이다.

| 항목 | 결정 | 이유 |
|------|------|------|
| **구현 범위** | 연산 엔진 코어만 (백엔드 API·DB·앱 UI 제외) | 데이터·디자인 없이 가능한 최대치. 알고리즘이 서비스의 핵심 가치 |
| **언어/스택** | 엔진 = **Python** / 백엔드 = NestJS(다음 단계) | 그래프·기하·PostGIS·수치 계산에 Python이 최적. 이번엔 순수 Python |
| **ML 모델** | **데이터 확보 후 착수** (지금은 결정론만) | 학습 데이터 부재. 단 그늘 계산은 기하이므로 포함 |
| **런타임 LLM** | 사용 안 함 (Claude API 제약 무관) | 핵심은 결정론 엔진. LLM 레이어는 로드맵 Phase 3 |

**핵심 설계 원칙 (기획서 3장 계승):** 안전 계산은 결정론적 규칙 엔진으로, 똑똑함이 필요한 곳에만 나중에 AI를 얹는다. 안전 판단을 LLM 환각에 맡기지 않는다.

## 1. 지금 만드는 것 — 연산 엔진 코어 5모듈 (M0~M4)

**대상 지역: 서울 송파구 전체.** OSM에서 송파구 보행 네트워크를 받아 그래프로 구축한다.

> **범위 조정(2026-07-08):** M5 프로필 개인화 · M6 게임화는 **별도 기획 중**이라 이번 구현에서 제외. 단, M2·M3의 파라미터(위험 임계값·비용 가중치)를 **주입식 인터페이스**로 설계해, 나중에 M5가 껍데기만 갈아끼우면 되도록 한다.

모두 순수 로직 + 조정 가능한 파라미터 + mock 데이터로 구현/검증한다. 실데이터는 나중에 동일 스키마로 끼워넣는다.

```
┌─────────────────────────────────────────────────────────┐
│ M0. 데이터 계약(스키마)  ← 모든 모듈의 공통 언어         │
└─────────────────────────────────────────────────────────┘
        │                    │
        ▼                    ▼
┌──────────────┐    ┌──────────────┐        ┌ ─ ─ ─ ─ ─ ─ ─ ┐
│ M1. 그늘 계산 │    │ M2. 위험지수  │◀ ─ ─ ─   M5. 개인화
│ (기하)        │───▶│ (결정론 공식) │        (파라미터 주입구만
└──────────────┘    └──────────────┘         열어둠, 구현 제외)
        │                    │                └ ─ ─ ─ ─ ─ ─ ─ ┘
        └──────────┬─────────┘
                   ▼
         ┌───────────────────┐
         │ M3. 라우팅 엔진    │
         │ (그래프+비용함수)  │
         └───────────────────┘
                   │
                   ▼
         ┌───────────────────┐        ┌ ─ ─ ─ ─ ─ ─ ─ ┐
         │ M4. 동적 리라우팅  │          M6. 게임화
         │ (공사·집회 회피)   │        (별도 기획, 제외)
         └───────────────────┘        └ ─ ─ ─ ─ ─ ─ ─ ┘
```

### M0. 데이터 계약 (스키마) — 최우선, 모든 모듈의 기반
수집이 진행 중이므로 **스키마가 곧 계약**이다. 여기서 정한 입출력 형식에 맞춰 실데이터가 들어온다.
- 반려견 프로필, 환경 관측, 보행 그래프(노드/엣지), POI, 동적 이벤트, 경로 결과, 게임화 상태
- 형식: Python `dataclass`/`pydantic` + 좌표는 WGS84, 지오메트리는 GeoJSON 호환
- 산출물: `schemas/` 모듈 + 각 스키마별 mock 샘플(fixtures)

### M1. 그늘/그림자 계산 (기하 — ML 아님, 지금 구현)
- **입력:** 태양 위치(고도·방위, 한국천문연구원 계산식/근사), 건물 높이·footprint(V-World), 가로수 위치·수관(트리맵)
- **출력:** 보행 그래프 각 엣지의 `shade_ratio`(0~1), 시간대별
- **알고리즘:** 태양 방위각 → 그림자 방향, 건물 높이·거리 → 그림자 길이/폴리곤, 엣지가 그림자에 덮이는 비율. 가로수는 반경 기반 그늘 원.
- **mock:** 샘플 건물/가로수 GeoJSON + 특정 시각(예: 7월 14시)

### M2. 위험지수 엔진 (결정론 공식)
- **입력:** 환경 관측(기온·습도·자외선·미세먼지) + **노면온도(입력값)** + 반려견 프로필
- **출력:** 위험지수 점수 + 신호등(초록/노랑/빨강) + 권장 산책 시간대 + 요소별 기여도
- **공식:** 열지수(기온+습도) + 노면온도(발바닥 화상) + 자외선 + 미세먼지를 정규화·가중합. 계절 모드(여름=그늘·노면 / 겨울=염화칼슘·빙판) 전환.
- **주의:** 노면온도 *추정 모델*(ASOS→RWIS 회귀)은 ML이라 데이터 후. 여기선 노면온도를 **입력 파라미터**로 받아 계산까지만.
- **mock:** 날씨 시나리오 × 견종 프로필 조합 → 예상 신호등을 assertion으로 검증

### M3. 라우팅 엔진 (그래프 + 비용함수) — 사장님이 콕 집은 핵심
- **입력:** 보행 그래프, 출발/목적지(또는 동네 순환), 위험지수 컨텍스트, 프로필
- **출력:** 경로(polyline/GeoJSON), 총 거리, 예상 시간, 그늘 비율, 경유 POI, 경고
- **비용함수:** `edge_cost = 시간 × (1 − w_shade·shade_ratio + w_hazard·위험 + w_공사 + w_집회 + w_교통)`, 안전 게이트로 위험 구간 hard-block
- **알고리즘:** A*/Dijkstra(수정 비용), 순환 경로는 목표 거리 근사 loop 생성
- **예상 시간:** 견종 보행 속도 × 거리(프로필 반영)
- **mock:** 샘플 OSM extract(한 동네) + 합성 환경·POI

### M4. 동적 이벤트 회피 & 자동 리라우팅 (결정론)
- **입력:** 활성 경로 + 신규 공사/집회 폴리곤(GeoJSON)
- **처리:** 경로 엣지가 폴리곤과 교차(shapely) → 해당 엣지 차단/고가중 → 재계산 → 변경 diff와 알림 문구 생성("앞쪽 집회로 경로를 변경했어요")
- **mock:** 경로 위에 놓이는 공사 폴리곤 샘플

### M5. 견종·반려견 프로필 개인화 (결정론 규칙)
- **입력:** 반려견 프로필(견종·나이·체중·털·단두종·지병)
- **출력:** 위험 임계값 세트 + 라우팅 가중치 세트 (M2·M3에 주입되는 파라미터 프로바이더)
- **규칙:** 단두종→열 가중↑, 소형견→겨울 저체온, 노령·비만→강도↓, 슬개골·관절→계단·점프 회피, 심장→거리·강도 제한
- **mock:** 대표 프로필 5종 → 파라미터 매핑 검증

### M6. 게임화 규칙 엔진 (결정론 규칙)
- **범위:** 미션(수집·완주·발견·지속·경쟁), 보상(수집형·실질형·특별), 레벨/XP, 스탬프·도감, 이코노미(일일 상한·시즌 리셋·검증 통과분만 지급)
- **★ 안전 게이트(필수):** 위험지수가 높은 날엔 보상 미션 자동 비활성/실내·단축 전환 → **M2와의 경계면이 핵심 검증 포인트**
- **mock:** 사용자 활동 이벤트 시퀀스 → 보상/레벨/게이트 상태 전이 검증

## 5. 검증 전략 (QA)

- **점진적 QA:** 각 모듈 완성 직후 검증. 경계면(예: 위험지수→안전 게이트, 그늘→라우팅 비용)을 양쪽에서 교차 비교.
- **정량 검증:** 파일 생성·경로 산출·상태 전이 등 객관 검증 가능 항목은 assertion으로 고정.
- **mock 시나리오:** 각 모듈당 정상 1 + 경계/에러 1 이상.
- **재현성:** 결정론이므로 같은 입력 → 같은 출력. 회귀 테스트로 고정.
