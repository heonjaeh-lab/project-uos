> 기준: `PawTrail_기획서_v3.pdf` 7대 기능 + 현재 완성된 연산 엔진(M0~M4) 출력. 작성 2026-07-08.

## 0. 디자인 방향 (브랜드 스타터)

**브랜드 성격:** "우리 개가 다칠까 봐"의 불안을 덜어주는 **안전 우선 · 다정한** 산책 동반자. 귀엽되 미덥게. 신호등(안전) 은유가 핵심 정체성.

- **팔레트(제안):** 배경 따뜻한 오프화이트(`#F5F4F0`)/다크 딥슬레이트(`#12140F`). 액센트는 **세이프 그린-틸**(`#0F766E`, 다크 `#5EEAD4`). 그늘=틸, 햇볕=앰버(`#E08A1E`)로 산책 테마 이중색. **신호등 시맨틱**은 액센트와 분리: green `#16A34A` / yellow `#D97706` / red `#DC2626`. POI 색: 동물병원 `#E11D48`·화장실 `#2563EB`·공원 `#16A34A`·급수대 `#0891B2`·펫샵 `#D97706`.
- **타이포:** 한글 본문은 시스템 산세리프(Pretendard/Apple SD Gothic Neo 계열) 권장. 숫자(거리·시간·지수)는 `tabular-nums`. 라벨은 약간의 자간.
- **톤&보이스:** 능동·구체. "지금 나가기 좋아요", "앞쪽 공사로 경로를 바꿨어요". 경고는 이유+대안. 진단처럼 단정하지 말 것(면책: 참고용).
- **UX 원칙(기획서):** 산책 중 안내는 **음성·진동 중심**(화면 안 봐도 됨), **사진 제출은 정지 후에만**, 위험한 날은 보상 미션 자동 축소(안전 게이트).
- **피해야 할 것:** 템플릿틱한 보라 그라데이션 히어로, 이모지 섹션 마커 남발, 전부 가운데 정렬. 지도·신호등·반려견 사진이 주인공.

| 컴포넌트 | 내용 | 엔진 데이터 |
|----------|------|-----------|
| **위험지수 신호등 배지** | 점수(0~100) + 신호등 dot(초/노/빨) + 주요인(heat/surface/uv/pm) + "부분데이터" 표기 | `RiskResult`(M2) |

| **지도 뷰** | 도로망 배경 + 경로(그늘=틸/햇볕=앰버 색상) + POI 마커 + 현재위치 + 출발/도착 | `RouteResult` polyline+segment shade, `POI` |
| **경로 요약 카드** | 거리·예상시간·평균그늘%·최대위험 신호등·경유 안전지점 수 | `RouteResult` |

## 2. 화면별 명세

### S1. 온보딩 · 반려견 프로필 등록  〔MVP〕
- **목적:** 개체별 안전 기준을 개인화하기 위한 최소 입력.
- **요소:** 사진 업로드, 이름, **견종**(검색), 나이, 체중, 털 길이(단/중/장), **단두종 여부**(토글), 체급, **지병 다중선택**(슬개골·관절·심장·비만…), 위치정보/사진 처리 동의.
- **데이터:** → `DogProfile`. (지금은 필드만; 개인화 로직 M5는 기획 중 — 입력값은 저장·표시)
- **상태:** 신규/수정, 다견 등록(여러 마리).
- **인터랙션:** 견종 검색 자동완성, 지병 선택 시 "산책 강도·시간 자동 조정" 안내 문구.

### S2. 홈 · 오늘의 산책  〔MVP〕
- **목적:** 앱 첫 화면. "지금 나가도 될까?"에 즉답.
- **요소:** 상단 **지금 시각 위험지수 신호등 배지**(송파구 실측) + 실측 환경 칩(기온·습도·PM·**강수확률**) / **산책 게이트 배너**(🐾 GO / ⚠️ CAUTION / 🚫 STOP) / **시간별 위험지수 스트립**(12시간, 매시간 색·점수 — 날짜 단위 아님) / "경로 추천받기" 큰 CTA / 케어 알림 요약 / 반려견 프로필 카드.
- **★ 산책 게이트(비 차단):** `walk_advisory` — **지금 비/눈이 오면 무조건 STOP**("지금 비가 와요, 산책을 미뤄주세요")하고 CTA를 막는다. 강수확률 ≥60% → CAUTION. 비가 없으면 열/미세먼지 위험지수 신호등을 따른다.
- **데이터:** `hourly_risk_series`(시간별 `RiskResult`+게이트) + `walk_advisory`(강수형태 PTY·강수확률 POP·강수량 PCP, 기상청 단기예보). **실측 연동 완료.**
- **상태:** 시간마다 go/caution/stop 변동. stop(비/red)일 땐 "산책 시작" 비활성 + 실내 대안 제안.
- **비고:** 프로토타입 → [오늘의경로_지도.html](오늘의경로_지도.html) 상단의 배지 + 시간별 스트립.

### S3. 경로 추천 · 지도  〔MVP · 핵심〕
- **목적:** 안전 경로를 지도로 제안하고 **사용자가 여러 안 중 하나를 골라** 진행. **앱의 심장.**
- **요소:**
  - **출발지 = 내 위치(GPS)** 자동 매칭(보행망 최근접 노드로 스냅) + 현재위치 마커. 목적지 입력 or **동네 순환**.
  - **다중 추천 경로**: `그늘 최대 / 균형 / 최단 거리` 카드(각 그늘%·거리·시간·안전지점 수). **지도에서 경로 클릭 or 카드 클릭으로 선택** → 선택 경로 하이라이트.
  - 선택 경로는 **구간별 그늘색**(햇볕=앰버→그늘=틸)으로 그려 어디가 뙤약볕인지 보이게.
  - 경로상 POI 마커+리스트, "경로에 추가", 계절 모드 토글, "산책 시작".
- **데이터:** `recommend_routes`(M3) → 라벨된 `RouteResult` 목록(그늘 많은 순, 중복 제거). GPS→`nearest_node`. **엔진 실데이터 구동 완료**.
- **상태:** 후보 1~3개 / "안전 경로 없음+사유"(모두 위험) / 로딩.
- **인터랙션:** 경로 선택, POI 부착, 그늘 우선도(shade_bonus) 조절, 재계산.
- **그늘 최적화 주의:** 그늘 여지는 **건물 높이**에 좌우됨. OSM 높이 태그가 24%뿐이라 면적 기반 휴리스틱으로 보강했고(그늘 커버리지 47→56%), **정확도는 V-World 실측 높이 연동 시 크게 향상**. 개활지(공원·대로) 구간은 본래 그늘이 적어 후보 간 차이가 작을 수 있음.
- **참고 시안:** [오늘의경로_지도.html](오늘의경로_지도.html) — 다중경로 선택·GPS·시간별 위험지수 실동작.

### S4. 경로 상세 · 산책 준비  〔MVP〕
- **목적:** 출발 전 경로 확인.
- **요소:** 경로 요약, **경유 편의/안전 지점 타임라인**(화장실·급수대·동물병원 등 순서+거리), 경고 목록(`warnings`), 예상 소요, 안내 방식(음성/진동) 설정, "시작".
- **데이터:** `RouteResult.pois_on_route`, `warnings`, `est_time_min`.

### S5. 실시간 산책 중 · 동적 리라우팅  〔Phase 2〕
- **목적:** 산책하며 화면 안 봐도 되는 안내 + 돌발 회피.
- **요소:** 현재 위치·경로 진행, **자동 리라우팅 알림**("앞쪽 공사로 경로를 바꿨어요"), 낯선 경로/교통량 경고, 다음 안전지점까지 거리, 음성·진동 안내, 크게 표시된 "일시정지/종료".
- **데이터:** `reroute_if_blocked`(M4) → 변경 경로 + `RouteWarning`(category=reroute, diff). **엔진 구동 완료**(공사·집회 폴리곤 교차 회피). 실 공사데이터는 서울 열린데이터광장 연동 예정.
- **UX:** 사진/제보는 **정지 후에만**. 안내 우선 음성.

### S6. 경로 공유 · 안심 공유  〔Phase 2〕
- **목적:** 가족·친구와 실시간 공유, 완주 자랑.
- **요소:** 실시간 위치·예상시간 공유("지금 어디쯤, 몇 분 남음"), **완주 인증 카드**(경로+사진, SNS 공유), 야간·혼자 산책 시 지정 연락처 위치 공유, 공유 대상·기간 동의/중단.
- **데이터:** `RouteResult` + 산책 진행. 법·규제: 위치공유 동의·언제든 중단.

### S7. 산책 기록 · 건강 리포트  〔Phase 2〕
- **목적:** 기록·습관·건강 관찰.
- **요소:** 산책 히스토리(거리·시간·경로 지도 썸네일), 배변 로그, **주간 활동 리포트**(거리·빈도·그늘 비율 추이), (후순위) 활동량·속도 변화 이상 시 **"수의 상담 권유" 참고 신호**(진단 아님, 면책 명시).
- **데이터:** 산책 로그(내부 축적). 건강 이상은 Phase 3.

### S8. 게임화 · 도감/미션/랭킹  〔MVP(기본)~Phase 2〕
- **목적:** 매일 나가고 싶게, 모으고 자랑하게.
- **요소:** **포토존 도감**(수집), 완주 스탬프·데일리/위클리 미션·레벨·연속 산책 스트릭, 지역 랭킹·구역 정복, 반려견 사진 콘테스트, 보상(배지·칭호·코스튬 / 포인트→쿠폰·병원할인 / 특별보상 쿠폰). **안전 게이트: 위험일 미션 자동 축소.**
- **데이터:** 게임화 상태(M6 — **별도 기획 중**, 이번 엔진 범위 밖). 안전 게이트는 위험지수(M2)와 연동.
- **비고:** 규칙 확정 후 스키마/엔진 추가 → 그때 화면 데이터 확정.

### S9. 제보 (게임화=데이터 엔진)  〔MVP〕
- **목적:** 지도 데이터(제보) 확보.
- **요소:** **위험요소 제보**(염화칼슘·유리·공사·빙판·말벌 — 위치+사진), 포토존 제보, 그늘/노면 체감 피드백, 사진 검증 상태(EXIF·GPS·타임스탬프+강아지 탐지, 얼굴·번호판 블러 안내).
- **데이터:** → 크라우드소싱(위험요소 태그, TTL). 라우팅 `hazards`로 반영. 사진 검증은 Phase 2.

### S10. 케어 알림 · 펫 맞춤 정보  〔Phase 2〕
- **목적:** 프로필+환경 결합 맞춤 안내.
- **요소:** 프로필 기반 주의(단두종 여름 열사병·소형견 겨울 저체온·노령/비만 강도↓), 지병 기반 경로 조정 안내(슬개골→계단 회피·심장→거리 제한), **케어 리마인더**(진드기·심장사상충·꽃가루·예방접종·구충 주기), 낯선 경로 안전 경고.
- **데이터:** `DogProfile` + `EnvObservation` + 위험지수. (개인화 로직 M5 기획 중)

### S11. 설정 · 프로필 관리 · 법적 고지  〔MVP〕
- **요소:** 다견 프로필 관리, 위치공유 동의·기간·중단, 알림 설정, **면책 고지**("경로는 참고용, 최종 판단은 보호자 책임. 건강 알림은 참고 신호이며 진단·처방 아님"), 개인정보 처리방침, 계정.

## 4. 엔진 연동 현황 (디자인 시 "실데이터로 채워지는" 부분)

- ✅ **지금 실데이터로 구동:** **시간별** 위험지수 + **산책 게이트/비 차단**(S2, `hourly_risk_series`/`walk_advisory`), **다중 추천 경로**+GPS 출발 매칭(S3, `recommend_routes`/`nearest_node`), 경로·거리·시간·구간별 그늘·경유 POI(S3/S4), 동적 리라우팅(S5), 실측 기온·습도·PM·**강수**(S2 칩), **V-World 실측 건물높이로 그늘 계산**(경로 그늘 최대 78%까지 — 휴리스틱 대비 대폭 향상).
- 🔜 **연동 예정:** 서울 열린데이터광장(가로수·급수대·공사) · 자외선 API(활용신청) · 노면온도(RWIS).
- 🧩 **별도 기획 중(엔진 미구현):** 게임화(S8) · 프로필 개인화 로직(S1/S10의 자동 조정) · 건강 이상(S7 후순위).

## 5. Claude로 각 화면 디자인하는 법
