# 조심해야댕 엔진 오케스트레이터

반려견 안전 산책 앱 **조심해야댕**의 연산 엔진 코어(M0~M4)를 에이전트 팀으로 개발한다. 안전 계산은 결정론, 데이터는 mock, 대상 지역은 송파구.

## 실행 모드: 에이전트 팀
모듈 경계면 조율이 잦아 팀 통신이 유리하다. 4명(architect, geo-routing-engineer, rules-engineer, qa-verifier)이 파일 기반 산출물 + 메시지로 협업한다.

## Phase 0: 컨텍스트 확인 (항상 먼저)
1. `engine/` 존재 여부로 실행 모드 판별:
   - `engine/` 없음 → **초기 구축** (Phase 1부터 전체)
   - `engine/` 있음 + 부분 수정 요청 → **부분 재실행** (해당 모듈 담당 에이전트만)
   - `engine/` 있음 + 새 요구 → 기존 코드 읽고 확장
2. `docs/구현계획.md`와 `_workspace/` 산출물을 읽어 진행 상황 파악.
3. `.venv` 존재·의존성 설치 확인. 없으면 먼저 설치.

## Phase 1: 스키마 (M0) — architect
- `engine/schemas/` + `requirements.txt` + `_workspace/00_architect_contracts.md` 생성.
- **게이트: 다른 에이전트는 스키마 확정 전 시작하지 않는다.**

## Phase 2: 병렬 빌드 — geo-routing-engineer ∥ rules-engineer
- geo-routing: M1 그늘(`shade-calculation`) → 안정화.
- rules: M2 위험지수(`risk-index`).
- 두 모듈은 독립이라 병렬. 각자 완료 시 `_workspace/`에 요약, QA에 알림.

## Phase 3: 라우팅 (M3) — geo-routing-engineer
- M1(그늘)·M2(위험지수)를 엣지 비용/게이트에 반영. `routing-engine` 스킬.
- 송파구 그래프는 `scripts/fetch_songpa_graph.py`로 1회 받아 캐시.

## Phase 4: 동적 리라우팅 (M4) — geo-routing-engineer
- 공사·집회 폴리곤 교차 → 재계산. M3 위에 구현.

## Phase 5: 점진적 QA — qa-verifier (각 Phase 직후 병행)
- 모듈 완성 즉시 검증. 특히 경계면(그늘→비용, 위험등급→게이트, 경로→리라우팅).
- `tests/` + `_workspace/qa_*.md`.

## 데이터 전달 프로토콜
- **파일 기반(주)**: `_workspace/{phase}_{agent}_{artifact}.md`에 산출물 요약. 코드는 `engine/`에 직접.
- **메시지 기반**: 경계면 shape 불일치·이슈는 SendMessage로 즉시 조율.
- 중간 산출물(`_workspace/`)은 보존(감사·재실행용).

## 에러 핸들링
- 1회 재시도 후 재실패 → 해당 결과 없이 진행하되 보고서에 누락 명시.
- OSM 다운로드 실패 → 캐시 확인 → 재시도 1회 → 실패 시 명확히 보고(그래프 없이는 M3/M4 불가).
- 경계면 불일치 → 삭제 금지, 양쪽 출처 병기 후 담당자 조율.

## 팀 크기
4명(빌더 3 + QA 1). 중규모(모듈 5개). 팀원당 집중 영역 명확.

## 이번 범위 제약 (모든 에이전트 준수)
- 언어 **Python**, 데이터 **mock**(그래프만 송파구 실 OSM), **결정론 한정**, **런타임 LLM 금지**.
- M5(개인화)·M6(게임화) 제외 — M2·M3에 파라미터 **주입구만** 열어둠.

## 테스트 시나리오
- **정상 흐름**: 스키마→그늘·위험지수→라우팅(그늘 우선·red 회피)→공사 폴리곤 투입→재계산 성공. QA 전 단계 통과.
- **에러 흐름**: OSM 다운로드 실패 → 캐시 폴백/명확한 보고. 출발-목적 사이 안전 경로 단절(모두 red) → 예외 아닌 "경로 없음+사유" 반환.

## 후속 작업
- 부분 수정 요청 시 해당 모듈 담당만 재호출(전체 재빌드 금지).
- 파라미터 튜닝은 `engine/schemas/params.py`에서.
- M5/M6 기획 확정 시 에이전트·스킬 추가하고 CLAUDE.md 변경 이력 갱신.
